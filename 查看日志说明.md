# 如何查看 Next.js 服务器日志

## 方法 1：在运行服务器的终端查看

1. 找到运行 `npm run dev:proxy` 的终端窗口
2. 查看该窗口中的输出，包括：
   - ✅ 编译成功信息（绿色）
   - ❌ 错误信息（红色）
   - 📝 请求日志
   - 🔍 堆栈跟踪（Stack Trace）

## 方法 2：重新启动服务器查看完整日志

如果找不到原来的终端窗口，可以：

1. **停止当前服务器**：
   - 在终端按 `Ctrl + C`（Mac/Linux）或 `Ctrl + Break`（Windows）
   - 或者关闭终端窗口

2. **重新启动服务器**：
   ```bash
   cd /Users/balala/个人资料/Blog/MyBlog
   npm run dev:proxy
   ```

3. **查看启动日志**：
   - 服务器启动时会显示编译信息
   - 如果有错误，会显示红色错误信息
   - 访问页面时，会显示请求日志

## 常见的错误日志格式

### 编译错误示例：
```
Error: Cannot find module 'xxx'
  at ...
```

### 运行时错误示例：
```
[Error] Internal Server Error
  at GET (app/api/price/route.ts:123:45)
  ...
```

### 请求日志示例：
```
GET /api/price?type=crypto&action=price&symbol=BTC 200 in 234ms
```

## 如何获取详细错误信息

1. **查看终端中的完整堆栈跟踪**
2. **查看浏览器控制台**（F12 → Console）
3. **查看 Network 标签**（F12 → Network → 点击失败的请求）
