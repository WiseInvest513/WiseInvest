# 如何提交代码

## 🚀 快速提交（3 步）

### 步骤 1: 添加所有更改

```bash
cd "/Users/balala/Persona/Blog/MyBlog"
git add .
```

### 步骤 2: 提交更改

```bash
git commit -m "fix: 修复类型错误和代码检查问题

- 修复 HistoricalPriceResult 重复导出错误
- 修复 ScheduledDataService 类型匹配问题
- 添加全局代码检查脚本
- 更新文档和 Git 操作指南"
```

### 步骤 3: 推送到 GitHub

**方法 A: 直接推送（如果网络正常）**

```bash
git push origin main
```

**方法 B: 使用代理推送（如果网络超时）**

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
git push origin main
```

## 📋 完整操作流程

### 1. 打开终端

按 `Command + Space`，输入 `Terminal`，按 Enter

### 2. 进入项目目录

```bash
cd "/Users/balala/Persona/Blog/MyBlog"
```

### 3. 检查当前状态（可选）

```bash
git status
```

### 4. 添加所有更改

```bash
git add .
```

### 5. 提交更改

```bash
git commit -m "fix: 修复类型错误和代码检查问题

- 修复 HistoricalPriceResult 重复导出错误
- 修复 ScheduledDataService 类型匹配问题
- 添加全局代码检查脚本
- 更新文档和 Git 操作指南"
```

### 6. 推送到 GitHub

**先试试直接推送**：

```bash
git push origin main
```

**如果超时，使用代理**：

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
git push origin main
```

## 💡 提交信息说明

### 提交信息格式

```
类型: 简短描述

详细说明（可选）
- 修复项 1
- 修复项 2
```

### 常用类型

- `fix`: 修复 bug
- `feat`: 新功能
- `docs`: 文档更新
- `refactor`: 代码重构
- `style`: 代码格式
- `test`: 测试相关

## 🔍 提交前检查（推荐）

在提交前，可以运行检查脚本：

```bash
# 运行全局检查
./pre-commit-check.sh

# 或单独检查类型
npx tsc --noEmit
```

## ⚠️ 常见问题

### Q: 提示 "nothing to commit"
**A**: 说明所有更改已经提交，直接推送即可：
```bash
git push origin main
```

### Q: 推送时要求输入用户名和密码
**A**: 
- 输入 GitHub 用户名
- 输入密码（如果启用了 2FA，使用 Personal Access Token）
- 生成 Token: https://github.com/settings/tokens

### Q: 推送超时怎么办？
**A**: 使用代理：
```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
git push origin main
```

### Q: 如何查看提交历史？
**A**: 
```bash
git log --oneline -10
```
